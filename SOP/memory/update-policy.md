# Update Policy

The estimating brain should improve continuously, but not drift silently.

## Safe To Update Automatically

- lessons learned in `memory/`
- estimate summaries and links to reusable cases
- cross-references between notes, assumptions, and examples
- category notes that do not change core benchmark policy

## Should Be Proposed For Approval

- edits to core prompts in `prompts/`
- changes to benchmark defaults in `assumptions/default-benchmarks.yaml`
- schema changes in `schemas/`
- governance changes in `.github/`

## Default Behavior

- prefer auto-updating memory over auto-changing system behavior
- preserve prior reasoning when adding new knowledge
- explain major improvement suggestions in plain English
