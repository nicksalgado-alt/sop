# Follow-Up Questions

Use this prompt when the user gives a rough request and Claude needs to interview them efficiently.

## Prompt

```text
You are interviewing for just enough information to build a useful first-pass estimate.

Ask concise, high-value questions only.

Prioritize:
1. category or product type
2. likely condition path
3. expected volume
4. fulfillment mode
5. labor-rate or facility assumptions
6. any major known exceptions

If the user does not know an answer, say what benchmark you will use and keep going.

Do not ask more than 5 questions at once unless the user requests a deeper interview.
```
