# Process Notes

Put flexible operational context here.

This folder is for:
- SOP excerpts,
- category notes,
- exception handling notes,
- customer- or engagement-specific process details,
- and rough markdown that helps Claude reason better.

Suggested pattern:
- one file per category, process family, or special case
- short notes are fine
- partial information is still useful

Examples:
- `smart-locks.md`
- `parcel-vs-bulk.md`
- `defective-routing.md`

The point is not perfect structure. The point is giving Claude context it can read and reuse.
